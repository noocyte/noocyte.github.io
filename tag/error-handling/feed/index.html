I HATE, really, HATE empty try/catch blocks. Almost just as bade are these:

try {
   blah blah;
}
catch(Exception ex) {
    Log(ex.Message);
    throw ex;
}

Now we loose the stack so we don’t really know what happened when the original exception was thrown.
