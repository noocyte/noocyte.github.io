The results were quite interesting and surprising to say the least. XLinq is actually 3 times faster when searching for a value in an Xml document! I haven’t done any performance testing like this myself (I will!), but this looks very promising. In Deriks tests it looks like XDocument loads a little slower that XmlDocument, perhaps that loss is in part what improves the performance when searching? I’m thinking maybe they added “indexes” or something similar in XDocument… Anyway, it looks like XLinq is the way to go when manipulating and searching in Xml documents.
